MP3 Tag Editor v2.0.9 Released

Posted on 1.2.2016

This build from the last build includes a lot of performance and method changes. I opted to use more Async to improve the speed performance. I also made the code cleaner, especially the save method, as before I had to loop at least 2 times through the save method for the cover to be applied.

Now, when the program scan's a folder for .mp3 file, It checks every file for corruption and excludes the file if it detects a corrupted file. I also made it so that when you open a song and close the program, it reopens that file the next time you launch the program.

Image opening has also been improved, as the program now only allows opening images with a 1:1 aspect ratio. If the image opened doesn't have a 1:1 aspect ratio, the program draws an overlay on top of the image container for 9 seconds, that indicates a non 1:1 aspect ratio image.

Also I'm happy to announce that the development of MP3 Tag Editor for the Universal Windows Platform is in works. The app will be availablefor PC and phones and Continuum enabled at launch.

MP3 Tag Editor Express Your Self Update

Posted on 23.1.2015

It took a while, but I succeeded. You can now customize the colors of the ui to your own needs. Just go to Tool/Setting and you will see this.

To change a color just click on the desired color rectangle. Then change the hue, saturation and brightness value by sliding the slider thumb or by manually inputting the values. The hue is measured from 0 to 360Â°. The saturation and brightness are measured from 0 to 100%. You can also just type in the 6 or 8 bit hex code. Just remember to press enter to apply the imputed value in the

You can also change the order by clicking on the combobox displayed in the top left picture and choosing the wished order. If you added a new file to the music folder just click on the refresh button display in the top middle picture. If you want the program to scan for files in sub folders just go to Tools/Setting and click on the toggle displayed in the top right picture. If you need to know how many .mp3 files do you have in a folder, there's a text block on the bottom right corner now. The program now also performers better as I got rid of the background worker and switched to a Async method instead. Settings are also now stored in the user congif file, rather than in .txt file in AppData, which your can now remove by typing in %appdata% in search and then going to DavixDevelop\MP3 Tag Edit and deleting UserInfo.txt if you have it.
